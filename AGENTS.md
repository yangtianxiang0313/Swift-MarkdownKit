# Project Rules

## First-Principles Refactor Rule (Highest Priority)

When doing refactors or fixes in this project:

1. Always start from first principles.
2. Always choose the best technical solution.
3. Do not optimize for minimal change.
4. Do not optimize for backward compatibility.
5. Do not provide migration-oriented compromises.
6. Prefer bold, potentially breaking corrections when they improve architecture and correctness.

This is a new project, so breaking changes are allowed and expected when they produce a better end-state.
